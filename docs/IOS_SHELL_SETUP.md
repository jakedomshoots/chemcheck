# iOS Shell Setup

This document describes how to build, run, and archive the ChemCheck iOS app. The native Xcode project is generated by Capacitor and lives in `ios/`.

## Prerequisites

- macOS with Xcode installed (latest stable version recommended)
- Xcode Command Line Tools: `xcode-select --install`
- An Apple Developer account (for TestFlight / App Store distribution)
- Node.js and npm (same versions used for the web app)

CocoaPods is **not** required for this project — the Capacitor plugins used here ship with `Package.swift` and use Swift Package Manager.

## Project structure

| Path | Description |
|------|-------------|
| `capacitor.config.json` | Capacitor configuration: app ID, plugins, background color, App-Bound Domains |
| `ios/App/App.xcodeproj` | Native Xcode project |
| `ios/App/App/Info.plist` | Native permissions, usage strings, App-Bound Domains |
| `ios/App/App/PrivacyInfo.xcprivacy` | Apple privacy manifest |
| `assets/icon.png` | Source app icon (1024×1024) |
| `assets/splash.png` | Source splash screen (2732×2732) |
| `scripts/ios-sim-run.sh` | Build and launch on the iOS Simulator |
| `scripts/ios-archive.sh` | Create an `.xcarchive` for TestFlight / App Store |

## Quick commands

```bash
# Sync web assets into the iOS project
npm run ios:sync

# Open the Xcode project
npm run ios:open

# Build and run on the iOS Simulator
npm run ios:sim

# Create an archive (signing required for distribution)
npm run ios:archive
```

## Running on the simulator

```bash
npm run ios:sim
```

The script:
1. Builds the production web bundle.
2. Syncs it into `ios/App/App/public`.
3. Builds the native app for the configured simulator.
4. Boots the simulator if needed.
5. Installs and launches the app.

Set `IOS_SIM_NAME` to choose a different device:

```bash
IOS_SIM_NAME="iPhone 15 Pro" npm run ios:sim
```

## Archiving for TestFlight / App Store

1. Make sure your Apple Developer team and signing certificates are configured in Xcode.
2. Update the version and build number in `ios/App/App.xcodeproj` if needed.
3. Run:

```bash
npm run ios:archive
```

4. Open Xcode Organizer and distribute the resulting `.xcarchive`:

```bash
npm run ios:open
```

Then choose **Window → Organizer**, select the archive, and click **Distribute App**.

## Updating icons or splash screen

Replace the source images:

- `assets/icon.png` — recommended 1024×1024
- `assets/splash.png` — recommended 2732×2732

Then regenerate the native assets:

```bash
npx @capacitor/assets generate --ios
npm run ios:sync
```

## Bundle ID, version, and build number

- **Bundle ID**: `com.chemcheck.app` (set in `capacitor.config.json` and Xcode)
- **Version**: edit `MARKETING_VERSION` in Xcode or the project settings
- **Build number**: edit `CURRENT_PROJECT_VERSION` in Xcode

Make sure the version and build number in Xcode match the values in App Store Connect before uploading.

## Permissions

The following permissions are declared in `ios/App/App/Info.plist`:

- Camera — for capturing before/after service photos
- Photo Library — for attaching existing photos
- Location — for future GPS check-in support
- Face ID — for future biometric auth support

Update the usage strings in `Info.plist` if your App Store copy changes.

## App-Bound Domains

`limitsNavigationsToAppBoundDomains` is enabled in `capacitor.config.json`. The allowed domains are listed in `ios/App/App/Info.plist` under `WKAppBoundDomains`:

- `clerk.chemcheck.xyz`
- `challenges.cloudflare.com`

These are WebView navigation domains, not every backend API host. `VITE_CONVEX_URL` must still point at the exact production Convex deployment in the release environment. If you add new authentication, payment, support, or API flows that navigate the WebView to another production domain, add only that exact production domain to both `capacitor.config.json` and `WKAppBoundDomains`.

## Privacy manifest

`ios/App/App/PrivacyInfo.xcprivacy` declares:

- Collected data types: email, phone number, photos/videos, crash data
- Accessed required-reason API: UserDefaults, reason `CA92.1`

Keep this file in sync with your public privacy policy (`public/privacy-policy.html`) and App Store Connect App Privacy answers. `plutil` validates plist syntax only; before upload, verify the accessed API categories and reasons against Apple's required-reason API schema in Xcode/App Store Connect.

## Troubleshooting

### `xcodebuild` cannot find simulators or says full Xcode is required

Make sure full Xcode is installed and selected, not only Command Line Tools:

```bash
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
xcodebuild -version
```

### Simulator build fails with signing errors

`scripts/ios-sim-run.sh` falls back to a no-sign simulator build automatically.

### Capacitor plugins are not reflected in the native project

Run:

```bash
npm run ios:sync
```

### Splash screen does not hide

The web app calls `SplashScreen.hide()` from `@capacitor/splash-screen` after the first render. If you change the root component, make sure that call remains.
